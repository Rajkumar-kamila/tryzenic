Le no-code est l'une des approches de programmation les plus en vogue actuellement. Sa simplicit\u00e9 et les nombreuses possibilit\u00e9s qu'il offre s\u00e9duisent de plus en plus d'entreprises et de d\u00e9veloppeurs web, notamment pour la r\u00e9alisation de POC (p<\/strong>roof o<\/strong>f c<\/strong>oncept) ou de MVP (minimum product viable). Citron Noir vous explique ce qu'est le no-code et \u00e9voque ses avantages ainsi que ses limites.<\/p>\n\n\n\n

No-code : la programmation informatique \u00e0 la port\u00e9e de (presque) tous<\/h2>\n\n\n\n

L'expression \u00ab no-code \u00bb est un anglicisme qui signifie litt\u00e9ralement \u00ab pas de code \u00bb en Fran\u00e7ais. Il sert \u00e0 d\u00e9signer une approche de d\u00e9veloppement<\/strong> dans laquelle l'utilisateur n'a pas besoin d'avoir des connaissances en programmation informatique pour concevoir un logiciel, un site web ou une interface m\u00e9tier. Tout ce dont il a besoin est une id\u00e9e claire des fonctionnalit\u00e9s qui seront utiles pour son entreprise ou son activit\u00e9. Les applications no-code <\/strong>offrent une large palette d'outils et de packages pr\u00eats \u00e0 l'emploi \u00e0 l'utilisateur. Ce dernier n'a donc plus besoin d'\u00e9crire la moindre ligne de code : interfaces et widgets prer\u00e9alis\u00e9s, modules \u00e0 glisser et d\u00e9poser pour cr\u00e9er de nouvelles fonctionnalit\u00e9s\u2026 Lorsqu'un environnement no-code int\u00e8gre la possibilit\u00e9 d'\u00e9diter le code source pour obtenir des interfaces ou applications personnalis\u00e9es et sur-mesure, on parle alors de low-code (peu de codes). Nous n'aimons pas tellement le terme no-code, sans aucune connaissance, vous aurez du mal \u00e0 vous plonger dans le no-code. L'architecture d'une application, la personnalisation, la cr\u00e9ation de base de donn\u00e9es ou de back-office demande quelques bases en d\u00e9veloppement. C'est notamment le cas en utilisant l'outil bravo studio<\/a>, que nous avons test\u00e9 de long, en large et en travers : Un outil no-code, mais relativement complexe, sans connaissance en d\u00e9veloppement informatique. C'est un exemple concret d'un outil commercialis\u00e9 comme no-code alors qu'en r\u00e9alit\u00e9 il se rapproche du low-code.<\/p>\n\n\n\n

Le no-code : quels sont ses avantages ?<\/h2>\n\n\n\n

Une prise en main extr\u00eamement simple<\/h3>\n\n\n\n

Les outils de d\u00e9veloppement no-code<\/strong> sont tr\u00e8s faciles \u00e0 prendre en main (sauf pour les low-code). Que ce soit pour la cr\u00e9ation d'un site web, d'une application mobile, ou pour l'ajout d'une fonctionnalit\u00e9 \u00e0 une interface existante, vous n'avez besoin que de connaissances basiques en informatiques pour obtenir le r\u00e9sultat souhait\u00e9. Un entra\u00eenement sur l'utilisation des fonctionnalit\u00e9s de l'outil choisi peut \u00eatre n\u00e9cessaire au d\u00e9but, mais aucune comp\u00e9tence en codage ou en d\u00e9veloppement d'interface web n'est n\u00e9cessaire.<\/p>\n\n\n\n

Une grande flexibilit\u00e9 pour l'adaptation aux changements<\/h3>\n\n\n\n

Des changements peuvent survenir \u00e0 n'importe quel moment dans la vie d'une entreprise pour diverses raisons : \u00e9volution des besoins des consommateurs, nouvelles tendances marketing\u2026 Avec le processus de d\u00e9veloppement classique, l'adaptation des diff\u00e9rents outils et canaux de l'entreprise \u00e0 ces changements prend du temps et n\u00e9cessite d'importantes ressources. Par contre, une plateforme no-code offre plus de flexibilit\u00e9<\/strong>. Des modifications\/ajouts peuvent \u00eatre rapidement effectu\u00e9es selon l'\u00e9volution des besoins m\u00e9tiers ou ceux de la client\u00e8le.<\/p>\n\n\n\n

Une ma\u00eetrise des co\u00fbts<\/h3>\n\n\n\n

D\u00e9velopper une application mobile ou un site web sur mesure n\u00e9cessite un investissement important. Pour une entreprise dont les ressources sont limit\u00e9es, c'est peut-\u00eatre un pari d'appeler une agence de communication<\/a> pour d\u00e9velopper son application mobile.<\/p>\n\n\n\n

Avec l'adoption d'un outil no-code<\/strong>, le budget allou\u00e9 au fonctionnement du service informatique peut \u00eatre drastiquement r\u00e9duit. Les ressources financi\u00e8res de l'organisation sont ainsi r\u00e9serv\u00e9es pour d'autres projets ne pouvant pas \u00eatre ex\u00e9cut\u00e9s avec cette approche.<\/p>\n\n\n\n

Toutefois, les fonctionnalit\u00e9s et la personnalisation sont tr\u00e8s limit\u00e9s. Ils sont tout simplement limit\u00e9s \u00e0 ce que propose la plateforme de cr\u00e9ation no-code.<\/p>\n\n\n\n

Quelles sont les limites du no-code ?<\/h2>\n\n\n\n

Malgr\u00e9 ces avantages, le no-code est une solution provisoire et\/ou limit\u00e9e en termes de personnalisation \/ utilisation.<\/p>\n\n\n\n

Des possibilit\u00e9s de conceptions limit\u00e9es<\/h3>\n\n\n\n

Pour certains besoins de personnalisations avanc\u00e9es, le no-code n'est pas la solution de conception la plus adapt\u00e9e<\/strong>. Dans ce genre de situation, il est pr\u00e9f\u00e9rable de se tourner vers un environnement (Framework) qui offre la possibilit\u00e9 \u00e0 l'utilisateur de modifier le code source et d'ajouter lui-m\u00eame des commandes.<\/p>\n\n\n\n

Difficult\u00e9 d'int\u00e9gration \u00e0 d'autres plateformes et difficult\u00e9s de migration<\/h3>\n\n\n\n

Certaines plateformes no-code proposent des composantes incompatibles avec d'autres sites. Il devient alors impossible de migrer vers un autre environnement en cas de changement ou lorsque les circonstances l'imposent.<\/p>\n\n\n\n

Vous n'\u00eates pas propri\u00e9taire du code source<\/h3>\n\n\n\n

Le no-code est tributaire d'un outil permettant de cr\u00e9er la plateforme. Le code source g\u00e9n\u00e9r\u00e9 par cet outil est d\u00e9pendant de la plateforme. Si celle-ci d\u00e9pose le bilan, vous ne pouvez pas r\u00e9cup\u00e9rer le code source de votre application, ni proposer de nouveaut\u00e9s.<\/p>\n\n\n\n

Peu de s\u00e9curit\u00e9 en place<\/h3>\n\n\n\n

Avec ces outils, les d\u00e9veloppeurs abandonnent une grande partie de leur contr\u00f4le \u00e0 des services tiers, ce qui peut repr\u00e9senter un risque de s\u00e9curit\u00e9 majeur. Un probl\u00e8me de plus en plus r\u00e9current, quels que soient les services et outils employ\u00e9s.<\/p>\n\n\n\n

Des plateformes peu cr\u00e9dibles<\/h3>\n\n\n\n

Bien que le no-code permette de cr\u00e9er des POC en peu de temps, les plateformes no-code sont souvent peu cr\u00e9dibles. En effet, lorsqu'il n'y a pas une r\u00e9elle \u00e9quipe de d\u00e9veloppement dans un projet, on rencontre souvent les m\u00eames probl\u00e9matiques :<\/p>\n\n\n\n

    \n
  • Design non adapt\u00e9 (parce que non r\u00e9alis\u00e9 par un UI designer)<\/li>\n\n\n\n
  • Performances limit\u00e9s<\/li>\n\n\n\n
  • Utilisation limit\u00e9s aux fonctionnalit\u00e9s : prendre en compte les retours des utilisateurs pour faire \u00e9voluer son projet est la base de tout projet web \/ mobile. Avec le no-code, il est souvent peu possible de d\u00e9velopper les fonctionnalit\u00e9s souhait\u00e9es<\/li>\n\n\n\n
  • Peu de valeur aupr\u00e8s des banques \/ investisseurs<\/li>\n\n\n\n
  • Donn\u00e9es peu exploitable (traitement des donn\u00e9es complexe)<\/li>\n<\/ul>\n\n\n\n
